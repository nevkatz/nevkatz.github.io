CSS Theme Switcher with VueJS

May 10, 2020

Switching themes with Vue

If you are building a website for a client, you may want to give your client an idea of how different themes look for a particular website element. In this case, it can help to have script that allows the client to dynamically switch themes and make a decision.

I recently found a way to build a quick theme switcher in VueJS below.

The interactive, which uses four themes from Canva’s website color schemes, allows you to switch themes using radio input elements, which are styled to look like tabs. When you press a tab, Vue works behind the scenes to change the CSS class of the output element.

Sample text is right here. The theme switcher is powered by VueJS.

We are pulling in the VueJS library from a CDN using a script tag:

<script src="https://npmcdn.com/vue/dist/vue.js"><script>

So for this interactive, there is no build step.

Step 1: Adding the root element

First, let’s add the main DOM element to our html and give it an ID of root.

Step 2: Adding the theme switcher and output

Now, let’s add an element for our theme switcher, and the container with id of output that will contain the elements with the resulting styles.

Step 3: Create a template element that will generate the theme options

Now we could just add some theme options using html radio buttons…

… but this gets repetitive, especially when add additional attributes to each input.

A less repetitive way to go about it is to create a list of themes and loop through them to create our inputs. We’ll lay the groundwork for that by adding a template element. The nice thinga about a template is that you can add Vue logic to it but it won’t show up in the DOM.

Step 4: Creating the Vue Object that holds the themes list

Let’s now start our Javascript file and create the Vue object that holds our themes list. Remember that by using that with a script tag mentioned earlier, we are already pulling in the Vue framework, which allows us to create the Vue object.

To make the Vue object act on our main root element, we will set its el property equal to #root.


 let vm = new Vue({
    el:'#root'
  });

Now let’s add a data object, which will hold our list of themes.


 let vm = new Vue({
    el:'#root',
    // add the data object.
    data: {}
  });

We can now add a themes property to our data object, which we can define as a list of four theme names.


 let vm = new Vue({
    el:'#root',
    // add the data object.
    data: {
       themes:['dark','modern','light','gemstone'],
    }
  });

Okay, so now we have a Vue object with four themes that is wired up to our HTML structure.

Step 5: Iterate through those themes

Let’s now go back to our HTML, where we will now use a Vue for-loop to iterate through those four themes and create our four options. To do this we’ll use v-for with the themes property, and use theme as the term for the current one in the loop.

What will we do with the theme each time? Why we’ll make our radio input element. In the input element below, notice that we have a colon : before value below, which allows us to reference the theme variable.

Step 6: Wire the input elements to Vue object

So we can generate our list of input elements now, but we ned to make it so that when you select one, it sets the current theme. So let’s create a variable called selected in our Vue object…


 let vm = new Vue({
    el:'#root',
    data: {
      themes:['dark','modern','light','gemstone'],
      // add the 'selected' property here.
      selected:''
    }
  });

…and then add a property called v-model on each input element, which will allow each one to set the theme by way of the selected property.

Step 7: Add an ID and a label to each input option

While we’re at it, let’s add an ID to each input and let’s give each input a label, which will actually be the visible element that we see.

Notice that the id attribute will also be the theme name, and the for attribute of the label will also be the theme name and therefore connect the label to its radio input element. We are using colons before each of these attributes so we can use that Vue goodness to reference the theme property.

Notice also that the label’s span element has a v-html property that is set equal to the theme element as well. This will allow the theme name to show up as visible text in each radio option.

Step 8: Show the result of selecting a theme

Now that we have some working options, let’s show the result of selecting a theme. We know that selecting an option will set the select variable on the Vue object to that option’s theme name, so now let’s pull that select variable into the output element.

We’ll do this in a couple ways: first, let’s set the class of the element using the colon syntax, and we will also add an <h1> header that prints the name of the theme.

Step 9: Add some additional elements that would be styled by the theme.

Now let’s add a <p> tag and a <button> element.

Step 10: Set first option as default theme.

And back in our Vue object, let’s add a way to set our first option as the default theme. To do this, let’s just set the selected property as the first element in the Vue object’s themes array.


  let vm = new Vue({
    el:'#root',
    data: {
      themes:['dark','modern','light','gemstone'],
      selected:''
    }
   });
   // set the default as the first element.
  vm.selected = vm.themes[0];

Finally, I’d like something to happen when you click that button - so let’s add a methods property to the Vue object. In methods, we’ll define a method called hello that handles the button click event with a simple “hello world.”

Step 11: Add a listener and handler for the button clicks.


  //define a new vue that grabs onto the root element.
  let vm = new Vue({
    el:'#root',
    data: {
      themes:['dark','modern','light','gemstone'],
      selected:''
    },
    methods: {
      hello:function() {
        alert('hello, world!')
      }
    }
   });
   // set the default as the first element.
   vm.selected = vm.themes[0];

And let’s add a click listener on the button using the v-on attribute.

Step 11: Add the CSS

We wouldn’t be able to change the style without CSS! This CSS has three sections:

  • general font style
  • the theme switcher styles
  • general styles for the output
  • specific styles for each theme

Let’s first add a general font family.

body {
  font-family: Arial;
}

Now let’s add the style for the switcher. Notice that I’m hiding the input elements and only showing the label elements so I can make the options look like tabs.


/* theme-switcher */
#theme-switcher {
  margin-bottom: 10px;
  border-bottom: 1px solid darkgrey;
}
/* hide the radio button element */
#theme-switcher input[type="radio"] {
  display: none;
}
/* style the label */
#theme-switcher label {
  padding: 10px;
  background-color: lightgrey;
  text-align: center;
  cursor: pointer;
  display: inline-block;
  margin-top: 0px;
  min-width: 50px;
  border-right: 1px solid darkgrey;
}
#theme-switcher label:last-child {
  border-right: 0px;
}
#theme-switcher input[type="radio"]:checked+label {
  background-color: dodgerblue;
  color: white;
}

Here I am just adding some general output styles.

/* styles that relate to the theme switcher */
#output {
  min-height: 130px;
  width: 100%;
}
#root h1 {
  font-size: 14px;
  margin: 0px;
  padding: 10px; 
  font-weight: 700;
}
#root p {
  margin: 0px;

  font-size: 13px;
}

  margin: 0px auto;
  display: block;
  border: 0px;
  padding: 8px;
  color:  #000;
  border-radius: 20px;
  cursor: pointer;
}

I hope this was a helpful introduction to the use of Vue JS. As you can see, you can put together a cool style switcher with a minimal amount of Javascript. You also end up defining a lot of your logic in your html, which connects it closely to your Vue object without the need for repetition.
